# Contributing Guide

感谢您对本项目的关注！

## 开发流程

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

## 代码规范

- 遵循 Go 官方代码规范
- 使用 `gofmt` 格式化代码
- 运行 `golangci-lint` 进行代码检查
- 编写单元测试覆盖新功能
- 更新相关文档

## 提交规范

提交信息应该清晰地描述改动内容：

- `feat:` 新功能
- `fix:` 修复bug
- `docs:` 文档更新
- `style:` 代码格式调整
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建过程或辅助工具的变动
